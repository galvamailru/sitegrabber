Правило отбора при парсинге (LLM)

С DEEPSEEK_API_KEY: на каждой странице LLM (1) решает, сохранять ли её в БД (should_collect и при заданном тексте правила — collection_rule_satisfied), (2) извлекает структуру товара/контента, (3) отдельным запросом выбирает, по каким исходящим ссылкам продолжать обход. Без ключа — эвристики и старая стратегия по путям. Правило сбора уточняет цель для обоих шагов.

Системный промпт чата витрины

Используется для POST /api/catalog-chat (виджет на опубликованном сайте). Задайте роль, тон, правила продажи каталога и услуг. Список товаров и инфо-страниц по-прежнему передаётся в сообщении пользователя; к системному промпту после публикации автоматически добавляется блок из настроек фильтров каталога (catalog_prompt_table), если он есть.
